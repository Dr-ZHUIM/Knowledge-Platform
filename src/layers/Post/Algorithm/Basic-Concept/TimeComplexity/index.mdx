## 时间复杂度分析

本文将讲解时间复杂度的定义以及计算方式

### 1. 什么是时间复杂度

时间复杂度是一个函数，它能够定性描述算法的运行时间。

时间复杂度能够辅助开发者开发者估算程序的运行时间，以此选择最优的算法。

时间复杂度的计算方式是：**在默认 CPU 的每个单元运行的消耗时间都相同的情况下，计算算法的操作单元数量**。

假设算法问题规模为 n， 那么操作单元数量使用函数 f(n) 来表示，随着数据规模的扩大， 算法执行的时间增长率与 f(n)的增长率相同，这就是算法的渐进时间复杂度，简称时间复杂度

即为： `O(f(n))`

### 2. 什么是 上界 O

算法导论提出： O(n) 中的 O 指的是上界，我们用 O 作为算法最极端情况下运行时间的最大值，在该例子中指的是：数据输入为 n 个，最极端情况就要运行 n 次，得到的时间复杂度就是 O(n)

### 3. O(logn)中的 log 是以什么为底？

如果一个算法的时间复杂度为 O(log^n)， 并不代表这是一个底数为 2 的对数，我们在这里忽略底数的描述

原因如下:

假设两个算法的时间复杂度，分别是 O($log_{2}{n}$) 和 O($log_{10}{n}$)，容易得出：`以2为底n的对数 = 以2为底10的对数 * 以10为底n的对数`

$$
	O(log_{2}{n}) = log_{2}{10} * O(log_{10}{n})
$$

那么同理，任意两个，以常数为底数的 n 的对数都可以拆解成相同的概念，而 $log_{i}{j}$ 是一个常数,在下文提到的化简方式中，会提到我们计算时间复杂度会忽略常数项系数。

$$
	logi^n = O(log_{i}{j} * log_{j}{n})
$$

至此我们得到以下左侧的函数，然后我们进一步忽略常数 i，得到最终的 $O(log^n)$

$$
    O(logi^n) => O(log^n)
$$
